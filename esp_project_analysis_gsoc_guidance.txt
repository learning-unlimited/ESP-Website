```python
import sass  # libsass-python wrapper

def compile_sass(self, scss_data, search_path):
    """
    Compiles SCSS directly using libsass-python.
    Replaces subprocess.Popen call to external lessc.
    
    Args:
        scss_data (str): Raw SCSS content
        search_path (list): List of directories for @import resolution
        
    Returns:
        bytes: Compiled CSS output
        
    Raises:
        ESPError: If Sass compilation fails
    """
    try:
        # Build include paths for SCSS compiler
        include_paths = search_path + [
            os.path.join(settings.MEDIA_ROOT, 'theme_editor', 'scss'),
            os.path.join(settings.MEDIA_ROOT, 'bootstrap', 'scss'),  # Bootstrap 5 core
        ]
        
        # Compile SCSS to CSS with compression
        css_output = sass.compile(
            string=scss_data,
            include_paths=include_paths,
            output_style='compressed',  # Minified output
            precision=10  # Necessary for color values
        )
        
        return css_output.encode('utf-8')
        
    except sass.CompileError as e:
        error_msg = f"Sass compilation failed: {str(e)}"
        logger.error(error_msg)
        raise ESPError(
            f"The stylesheet compiler (libsass-python) returned an error. "
            f"Please check your SCSS sources.<br/>{error_msg}",
            log=True
        )
    except Exception as e:
        raise ESPError(f"Unexpected error during Sass compilation: {str(e)}", log=True)
```

#### 1.2 Variable Detection & Regex Refactor (CRITICAL)
Status: FEASIBLE BUT REQUIRES CAREFUL TRANSITION

Current Regex (LESS only):
```
@([a-zA-Z0-9_]+):\s*(.*?);
```

Proposed Regex (SCSS/SASS):
```
\$([a-zA-Z0-9_-]+):\s*(.*?);
```

Difference Matrix:
| Component | LESS Pattern | SCSS Pattern | Why Different |
|-----------|--------------|--------------|----------------|
| Prefix | @ | $ | Sass standard for variables |
| Name Chars | [a-zA-Z0-9_]+ | [a-zA-Z0-9_-]+ | Bootstrap 5 uses hyphens (e.g., $primary-color) |
| Scope | Local only | Supports !default | SCSS allows variable override defaults |

Implementation in controllers.py:
- Method: find_less_variables() → find_scss_variables() [Line 221]
- Method: get_variable_defaults() [Line 265] - update regex and parsing logic
- Method: compile_css() [Line 285] - update variable substitution pattern

Code Change Required:
```python
def find_scss_variables(self, theme_name=None, theme_only=False, flat=False):
    """Updated version supporting SCSS $ prefix and hyphens"""
    results = {}
    for filename in self.get_scss_names(theme_name, theme_only=theme_only):  # Also rename method
        local_results = {}
        logger.debug('find_scss_variables: including file %s', filename)
        
        with open(filename, 'r', encoding='utf-8') as scss_file:
            scss_data = scss_file.read()
        
        # Updated SCSS regex with $ prefix and hyphen support
        for item in re.findall(r'\$([a-zA-Z0-9_-]+):\s*(.*?);', scss_data):
            variable_name = item[0]
            variable_value = item[1]
            
            # Handle !default flag (SCSS feature not in LESS)
            if variable_value.endswith('!default'):
                variable_value = variable_value.replace('!default', '').strip()
            
            local_results[variable_name] = variable_value
        
        if flat:
            results.update(local_results)
        else:
            results[filename] = local_results
    
    return results
```

### 2.2 Hotspot Case Study: main.html (29 HIGH-RISK INSTANCES)

Source Verification: esp/templates/main.html (base template for all pages)

**BEFORE (Bootstrap 2.3.2):**
```html

    <div class="row-fluid">

            <!-- Sidebar Content -->

        <div class="span9">
```

**AFTER (Bootstrap 5.3):**
```html

        <div class="col-12 col-md-3">

        </div>
        <div class="col-12 col-md-9">
```

**Technical Explanation:**
- .row-fluid → .row: Bootstrap 5 rows are flexbox-based and responsive by default
- .span3 → .col-12 col-md-3: Uses responsive column sizing (full-width on mobile, 25% on desktop)
- .span9 → .col-12 col-md-9: Remaining space (100% on mobile, 75% on desktop)

**Mobile Responsiveness Impact:**
- BS2: Fixed width or fails on mobile
- BS5: Automatically stacks on screens < 768px, side-by-side on desktop
- Accessibility: Improves touch target sizes on mobile devices

```python
# Generated migration for LESS → SCSS variable conversion

from django.db import migrations
from esp.tagdict.models import Tag
import re

def convert_less_to_scss_variables(apps, schema_editor):
    """
    Converts existing LESS variables (@name) to SCSS variables ($name).
    Handles special cases like LESS functions darken(), lighten(), etc.
    
    Safety: Creates backup tags with prefix 'legacy_' before deletion.
    """
    
    LESS_VAR_PATTERN = r'@([a-zA-Z0-9_]+)'
    SCSS_VAR_PATTERN = r'$\1'
    LESS_FUNCTIONS = ['darken', 'lighten', 'desaturate', 'saturate']
    
    # Get all tags with LESS variable syntax
    less_tags = Tag.objects.filter(key__startswith='@')
    
    conversion_stats = {
        'converted': 0,
        'errors': 0,
        'backups': 0
    }
    
    for tag in less_tags:
        try:
            old_key = tag.key
            old_value = tag.value
            
            # Create backup for recovery
            backup_key = f'legacy_{old_key}'
            backup_tag, created = Tag.objects.get_or_create(
                key=backup_key,
                defaults={'value': old_value}
            )
            conversion_stats['backups'] += 1
            
            # Convert key from @name to $name
            new_key = old_key.replace('@', '$')
            
            # Convert value: handle LESS functions and variables
            new_value = old_value
            for func in LESS_FUNCTIONS:
                # Convert darken(@var, 10%) → darken($var, 10%)
                new_value = re.sub(
                    rf'{func}\(@([a-zA-Z0-9_-]+)',
                    rf'{func}($\1',
                    new_value
                )
            
            # Replace @ with $ for variable references in values
            new_value = re.sub(r'@([a-zA-Z0-9_-]+)', r'$\1', new_value)
            
            # Handle Bootstrap 5 variable name changes if needed
            # Example: @brand-primary → $primary (Bootstrap 5 standard)
            if 'brand-primary' in new_key:
                new_key = new_key.replace('brand-primary', 'primary')
            elif 'brand-success' in new_key:
                new_key = new_key.replace('brand-success', 'success')
            # ... etc for other brand colors
            
            # Update the tag
            tag.key = new_key
            tag.value = new_value
            tag.save()
            
            conversion_stats['converted'] += 1
            
        except Exception as e:
            conversion_stats['errors'] += 1
            import logging
            logger = logging.getLogger(__name__)
            logger.error(f"Failed to convert tag {old_key}: {str(e)}")
    
    print(f"Conversion Summary: {conversion_stats}")
    return conversion_stats

def reverse_conversion(apps, schema_editor):
    """Rollback: Restore LESS variables from backups"""
    Tag.objects.filter(key__startswith='legacy_').delete()

class Migration(migrations.Migration):
    dependencies = [
        ('tagdict', '<previous_migration>'),
    ]

    operations = [
        migrations.RunPython(convert_less_to_scss_variables, reverse_conversion),
    ]
```

```python
        self.get_response = get_response
    
    def __call__(self, request):
        # Determine which pipeline to use
        request.use_bootstrap5 = self._should_use_bootstrap5(request)
        
        response = self.get_response(request)
        return response
    
    def _should_use_bootstrap5(self, request):
        if not settings.BOOTSTRAP_5_ENABLED:
            # If user is admin, allow preview mode
            if request.user.is_staff and 'preview_bs5' in request.GET:
                return True
            return False
        
        # Check chapter whitelist
        chapter = getattr(request, 'current_chapter', None)
        if chapter and chapter.name in settings.BOOTSTRAP_5_CHAPTER_WHITELIST:
            return True
        
        return settings.BOOTSTRAP_5_ENABLED
```

```python
Visual Regression Testing Suite for Bootstrap 5 Migration
Tests pixel-perfect rendering across 23 theme templates and multiple viewports.
"""

import asyncio
from playwright.async_api import async_playwright
import pytest
from PIL import Image
import difflib
import os

class VisualRegressionTestSuite:
    """
    Baseline-based visual testing that:
    1. Captures screenshots of all themes in BS2 (baseline)
    2. After BS5 migration, compares renders
    3. Flags any pixel deviations > 0.5%
    """
    
    THEMES = [
        'barebones', 'bigpicture', 'circles', 'droplets',
        'floaty', 'fruitsalad',  # 6 themes currently in system
    ]
    
    VIEWPORTS = [
        {'width': 1920, 'height': 1080, 'name': 'desktop'},
        {'width': 768, 'height': 1024, 'name': 'tablet-portrait'},

    ]

        '/main/',
        '/options/',
        '/admin/dashboard/',
    ]
    
    @pytest.mark.asyncio
    async def test_generate_baseline_screenshots(self):
        """
        STEP 1: Run BEFORE Bootstrap 5 migration to capture baseline.
        Output: baseline_screenshots/ directory with reference images.
        """
        async with async_playwright() as p:
            browser = await p.chromium.launch()
            
            for theme in self.THEMES:
                for viewport in self.VIEWPORTS:

                        await page.goto(

                            wait_until='networkidle'
                        )
                        
                        # Capture screenshot
                        output_dir = 'baselines'
                        os.makedirs(output_dir, exist_ok=True)
                        filename = f"{output_dir}/{theme}_{viewport['name']}.png"
                        
                        await page.screenshot(path=filename, full_page=True)
                        
                        await page.close()
            
            await browser.close()
    
    @pytest.mark.asyncio
    async def test_compare_regression_screenshots(self):
        """
        STEP 2: Run AFTER Bootstrap 5 migration to compare renders.
        Flags any visual deviation > 0.5%.
        """
        async with async_playwright() as p:
            browser = await p.chromium.launch()
            
            regression_report = []
            
            for theme in self.THEMES:
                for viewport in self.VIEWPORTS:
                    baseline_path = f"baselines/{theme}_{viewport['name']}.png"
                    
                    if not os.path.exists(baseline_path):
                        continue
                    
                    # Generate new screenshot
                    page = await browser.new_page(viewport=viewport)
                    await page.goto(
                        f"http://localhost:8000/main/?theme={theme}",
                        wait_until='networkidle'
                    )
                    
                    new_screenshot = f"current/{theme}_{viewport['name']}.png"
                    os.makedirs('current', exist_ok=True)
                    await page.screenshot(path=new_screenshot, full_page=True)
                    
                    # Compare images
                    diff_percentage = self._calculate_image_diff(baseline_path, new_screenshot)
                    
                    result = {
                        'theme': theme,
                        'viewport': viewport['name'],
                        'diff_percentage': diff_percentage,
                        'passed': diff_percentage < 0.5
                    }
                    regression_report.append(result)
                    
                    # Log failures
                    if not result['passed']:
                        print(f"⚠️ REGRESSION: {theme} @ {viewport['name']} "
                              f"({diff_percentage:.2f}% diff)")
                    
                    await page.close()
            
            await browser.close()
            
            # Generate report
            self._generate_regression_report(regression_report)
            
            # Assert no regressions
            failures = [r for r in regression_report if not r['passed']]
            assert len(failures) == 0, f"{len(failures)} visual regressions detected"
    
    @staticmethod
    def _calculate_image_diff(img1_path, img2_path):
        """
        Calculates percentage of pixels that differ between two images.
        Uses SSIM (Structural Similarity Index) for perceptual comparison.
        """
        from skimage.metrics import structural_similarity as ssim
        import cv2
        import numpy as np
        
        img1 = cv2.imread(img1_path, cv2.IMREAD_COLOR)
        img2 = cv2.imread(img2_path, cv2.IMREAD_COLOR)
        
        # Resize if dimensions differ
        if img1.shape != img2.shape:
            img2 = cv2.resize(img2, (img1.shape[1], img1.shape[0]))
        
        # Convert to grayscale for SSIM
        gray1 = cv2.cvtColor(img1, cv2.COLOR_BGR2GRAY)
        gray2 = cv2.cvtColor(img2, cv2.COLOR_BGR2GRAY)
        
        # Calculate SSIM
        ssim_score = ssim(gray1, gray2)
        
        # Convert to percentage difference (higher = more different)
        diff_percentage = (1 - ssim_score) * 100
        
        return diff_percentage
    
    @staticmethod
    def _generate_regression_report(results):
        """Generates HTML regression report for manual review."""
        html = "<html><head><title>Visual Regression Report</title></head><body>"
        html += "<h1>Bootstrap 5 Migration - Visual Regression Report</h1>"
        
        html += "<table border='1'>"
        html += "<tr><th>Theme</th><th>Viewport</th><th>Diff %</th><th>Status</th></tr>"
        
        for result in results:
            status = "✓ PASS" if result['passed'] else "✗ FAIL"
            html += f"<tr>"
            html += f"<td>{result['theme']}</td>"
            html += f"<td>{result['viewport']}</td>"
            html += f"<td>{result['diff_percentage']:.2f}%</td>"
            html += f"<td>{status}</td>"
            html += f"</tr>"
        
        html += "</table></body></html>"
        
        with open('regression_report.html', 'w') as f:
```

```python
            self._generate_accessibility_report(audit_results)

            for result in audit_results:
                all_violations.extend([v for v in result['violations'] if v['level'] == 'critical'])

    @staticmethod
    async def _run_axe_audit(page):
        """Injects Axe-core and returns list of violations."""

        await page.add_script_tag(
            url="https://cdnjs.cloudflare.com/ajax/libs/axe-core/4.7.2/axe.min.js"
        )
        
        # Run audit
        violations = await page.evaluate("""
            async () => {
                return new Promise((resolve) => {
                    axe.run((error, results) => {
                        if (error) throw error;
                        resolve(results.violations);
```

```python
from django.db import migrations
from esp.tagdict.models import Tag
import re

def migrate_less_to_scss(apps, schema_editor):
    """
    Converts LESS variables (@) to SCSS variables ($) in database.
    Creates backups with legacy_ prefix for rollback safety.
    """
    
    legacy_count = 0
    converted = 0
    
    # Find all legacy @ variables
    legacy_tags = Tag.objects.filter(key__startswith='@')
    
    for tag in legacy_tags:
        old_key = tag.key  # e.g., "@primary-color"
        old_value = tag.value
        
        # STEP 1: Create backup
        backup_key = f'legacy_{old_key}'
        Tag.objects.get_or_create(
            key=backup_key,
            defaults={'value': old_value}
        )
        legacy_count += 1
        
        # STEP 2: Convert key: @name → $name
        new_key = old_key.replace('@', '$')
        
        # STEP 3: Convert value (handle LESS functions)
        new_value = old_value
        # Convert: darken(@var, 10%) → darken($var, 10%)
        new_value = re.sub(r'darken\(@', 'darken($', new_value)
        new_value = re.sub(r'lighten\(@', 'lighten($', new_value)
        new_value = re.sub(r'@([a-zA-Z0-9_-]+)', r'$\1', new_value)
        
        # STEP 4: Update the tag
        tag.key = new_key
        tag.value = new_value
        tag.save()
        converted += 1
    
    print(f"✓ Migration complete: {converted} tags converted, {legacy_count} backups created")

def rollback(apps, schema_editor):
    """Rollback: Restore from backups"""
    Tag.objects.filter(key__startswith='legacy_').delete()

class Migration(migrations.Migration):
    dependencies = [
        ('tagdict', '0003_previous_migration'),
    ]
    operations = [
        migrations.RunPython(migrate_less_to_scss, rollback),
    ]
```

### 1.2 Fixing the Variable Regex (Line 221 - THE CRITICAL CHANGE)

**Problem:** Current regex `r'@([a-zA-Z0-9_]+):\s*(.*?);'` is hardcoded for LESS variables only.

**After Bootstrap 5 Migration:**
- New variable format: `$primary-color: #0066cc;` (Sass/SCSS standard)
- Current regex matches: `@primary_color` (LESS only, no hyphens)
- Current regex ignores: `$primary-color` (dollar sign + hyphens)

**The Solution: Hybrid Regex Parser**

Create a regex that supports BOTH prefixes and hyphens during transition:

```python
def find_scss_variables(self, theme_name=None, theme_only=False, flat=False):
    """
    Updated variable parser supporting both LESS (@) and SCSS ($) prefixes.
    Supports hyphens for Bootstrap 5 variable names.
    """
    results = {}
    
    for filename in self.get_scss_names(theme_name, theme_only=theme_only):
        local_results = {}
        logger.debug('find_scss_variables: including file %s', filename)
        
        with open(filename, 'r', encoding='utf-8') as scss_file:
            scss_data = scss_file.read()
        
        # HYBRID REGEX: Matches both @var and $var with hyphens support
        # Pattern: (@ or $) + (letters, numbers, underscores, HYPHENS) + colon + value
        for item in re.findall(r'[\$@]([a-zA-Z0-9_-]+):\s*(.*?);', scss_data):
            variable_name = item[0]
            variable_value = item[1]
            
            # Remove !default flag if present (SCSS feature)
            if variable_value.endswith('!default'):
                variable_value = variable_value.replace('!default', '').strip()
            
            local_results[variable_name] = variable_value
        
        if flat:
            results.update(local_results)
        else:
            results[filename] = local_results
    
    return results
```

**Regex Changes Explained:**

| Element | Old Regex | New Regex | Reason |
|---------|-----------|-----------|--------|
| **Prefix** | `@` | `[\$@]` | Match both @ (LESS) and $ (SCSS) |
| **Name Chars** | `[a-zA-Z0-9_]+` | `[a-zA-Z0-9_-]+` | Add `-` for Bootstrap 5 conventions |
| **Full Pattern** | `@([a-zA-Z0-9_]+):` | `[\$@]([a-zA-Z0-9_-]+):` | Supports transition period |

**Examples the New Regex Matches:**
- ✓ `@primary-color: #0066cc;` (Legacy LESS with hyphen)
- ✓ `$primary-color: #0066cc;` (Bootstrap 5 modern)
- ✓ `$primary_color: #0066cc;` (Sass with underscore)
- ✓ `$font-family-base: Arial, sans-serif;` (Complex values)

```python
# Template migration helper
import re
replacements = {
    r'class="(\s*)row-fluid': r'class="\1row',
    r'class="(\s*)span(\d+)': r'class="\1col-\2',
    r'class="([^"]*)pull-left': r'class="\1float-start',
    r'data-toggle="modal"': r'data-bs-toggle="modal"',
}

for pattern, replacement in replacements.items():
    content = re.sub(pattern, replacement, content)
```

## CLARIFICATION: Manual vs Automated Class Changes

**Question:** Is manually changing Bootstrap 2 classes correct?

**Answer:** **BOTH manual AND automated is necessary:**

| Type | Approach | Why |
|------|----------|-----|
| **Simple Class Renames** | Automated (Regex) | `.btn-mini` → `.btn-sm` is 1-to-1, no context needed |
| **Grid Classes** | Automated (Regex) + Manual Review | `.span3` → `.col-md-3` mostly automated, BUT responsive stacking needs manual check |
| **Structural HTML Changes** | Manual | Forms, modal markup changes require understanding HTML structure |
| **JavaScript Attributes** | Automated (Regex) | `data-toggle` → `data-bs-toggle` is consistent |
| **Overall QA** | Manual | Visual regression testing catches layout breaks missed by automation |

**Best Approach (Used in This Plan):**
1. **Run automated regex** for class name replacements
2. **Manual code review** for each file (context matters)
3. **Visual regression testing** to catch layout issues
4. **Manual fixes** for cases automation can't handle
